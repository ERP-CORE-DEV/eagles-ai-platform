MCP intelligence layer

The frontend-catalog-mcp server exposes 12 tools over stdio. Any Claude Code or MCP-compatible client can query the catalog at runtime.

ToolPurpose
catalog_search_patternFuzzy search over the component manifest by free-form query.
catalog_component_metadataExact lookup: file, tokens used, story parity.
catalog_audit_a11yRun the gate regex pack against an arbitrary TSX snippet.
catalog_compare_systemsCanonicalized comparison against 8 external systems.
catalog_generate_from_specTemplate substitution from a validated component spec.
catalog_token_playgroundApply token overrides, return diff + regenerated CSS.
catalog_list_tokensStructured dump of all W3C design tokens.
catalog_suggest_tokenSemantic role → token lookup.
catalog_get_componentSource read by name.
catalog_get_baselineBaseline layer (BASELINE-L1..L8) markdown.
catalog_get_adr_decisionADR decision lookup by number.
catalog_lint_snippetFind raw color literals and suggest replacements.

Demo — an LLM scaffolds a dashboard

$ claude mcp list
frontend-catalog: stdio @eagles-ai-platform/frontend-catalog-mcp

> Scaffold me a recruitment dashboard using the EAGLES catalog.

• catalog_search_pattern({ query: "dashboard" })
  → 48 candidates ranked. Top: KanbanBoard, DataTable, Stat, MetricCard.

• catalog_component_metadata({ name: "KanbanBoard" })
  → 29 tokens, has CSS, has Story. Safe to import.

• catalog_compare_systems({ target: "component", name: "DataTable" })
  → 8/8 systems cover DataTable. EAGLES ships its own, gate-clean.

• catalog_generate_from_spec({ name: "RecruitmentHeader", category: "layout", ... })
  → 3 files emitted, token-compliant, DarkMode story included.

Dashboard scaffolded. All components gate-pass. Light + dark ready.
