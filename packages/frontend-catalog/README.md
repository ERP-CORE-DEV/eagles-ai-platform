# @eagles-ai-platform/frontend-catalog

The first AI-queryable design system. 48 production-gated React components, W3C DTCG tokens, light + dark parity, and a 12-tool MCP intelligence layer built for LLM agents that scaffold real apps.

## What is this?

A design system catalog designed to be consumed by LLMs via MCP, not browsed by humans via a docs site. The companion package [`@eagles-ai-platform/frontend-catalog-mcp`](../frontend-catalog-mcp) exposes 12 stdio tools so any Claude Code or MCP-compatible client can:

- Search the catalog by free-form query
- Inspect component metadata (tokens used, story parity, file path)
- Audit arbitrary TSX snippets against the gate's regex pack
- Compare coverage across 8 external design systems (Carbon, Primer, Spectrum, Material 3, Fluent, Ant, Chakra, shadcn/ui)
- Generate new components from a locked template via a validated spec
- Preview token overrides and get back a diff + regenerated CSS

Every component:
- Ships with `.tsx` + `.css` + `.stories.tsx`
- Uses CSS variables exclusively (no hex, rgb, hsl, named colors)
- Exports `Default` + `DarkMode` stories (gate-enforced parity)
- Is function-component (no `React.FC`)
- Passes static a11y checks (no `<div onClick>`, icon-only buttons require `aria-label`, inputs require `id` or `aria-label`)

## The gate

`scripts/gate.mjs` is a zero-dependency 180-line script that enforces:

1. **Token-only CSS** — regex rejects `#hex`, `rgb(`, `hsl(`, named colors
2. **No React.FC** — function components only
3. **No inline `style={{...}}`** — single-brace `style={cssVarsConst}` allowed
4. **No Tailwind arbitrary values** (`className="p-[12px]"`)
5. **A11y** — icon-only buttons need `aria-label`, inputs need `id`/`aria-label`, no `<div onClick>`
6. **Dark-mode story parity** — every `*.stories.tsx` must export `Default` and `DarkMode`

Run it before every commit:

```bash
node scripts/gate.mjs                # lint only
node scripts/gate.mjs --manifest     # also write components.manifest.json
```

## Build

```bash
pnpm build:tokens         # style-dictionary → dist/tokens.css + .js + tailwind preset
pnpm build:storybook      # static Storybook → dist/storybook/
pnpm build                # both
pnpm lint                 # tsc --noEmit
```

## Dark mode

Set `data-theme="dark"` on `<html>` (or any ancestor) and every component inherits the dark palette via the W3C `$extensions.modes.dark` token overrides.

```tsx
document.documentElement.setAttribute('data-theme', 'dark');
```

## MCP intelligence layer

See the companion package [`frontend-catalog-mcp`](../frontend-catalog-mcp/README.md). Twelve tools total:

| Tool | Purpose |
|---|---|
| `catalog_search_pattern` | Fuzzy search over the component manifest |
| `catalog_component_metadata` | Exact lookup: file, tokens used, story parity |
| `catalog_audit_a11y` | Run the gate's regex pack against a TSX snippet |
| `catalog_compare_systems` | Canonicalized comparison across 8 external systems |
| `catalog_generate_from_spec` | Template substitution from a validated spec |
| `catalog_token_playground` | Apply token overrides, return diff + regenerated CSS |
| `catalog_list_tokens` | Structured dump of all 138 W3C tokens |
| `catalog_suggest_token` | Semantic role → token path |
| `catalog_get_component` | Source read by name |
| `catalog_get_baseline` | BASELINE-L1..L8 markdown |
| `catalog_get_adr_decision` | ADR decision lookup |
| `catalog_lint_snippet` | Find raw color literals, suggest token replacements |

## Cross-system comparison

`data/systems/*.json` in the MCP package contains flattened token + component data from 8 public design systems (515 tokens, 235 components). The `catalog_compare_systems` tool canonicalizes aliases (e.g., `dropdown` → `select`, `blue.60` → `color.primary`) so agents can answer "does Carbon have a DataTable?" or "how does Material 3 represent surface color?".

Ingested systems (all MIT or Apache-2.0):
- IBM Carbon
- GitHub Primer
- Adobe Spectrum
- Google Material 3
- Microsoft Fluent 2
- Ant Design
- Chakra UI
- shadcn/ui
